What it does

The official UPPCL SMART portal stops at "today's balance." This project goes further.

⚡

Live balance + runway

Auto-fallback chain ensures a trustworthy balance number. Fitness-ring gauge shows days remaining at current burn.

📊

Anomaly detection

Flags days where consumption is >1.5σ above the 30-day average, with a panel explaining the math.

💰

Cost breakdown

Effective ₹/unit trend, stacked charge composition, tariff slab usage, and subsidy YTD — all computed from the raw billing data.

🔋

Recharge sweet-spot

Interactive sliders recommend amount × frequency for a target runway, based on your actual historical burn rate.

🛡️

Meter health

Reading reliability (Actual vs Estimated), 365-day data-gap calendar, peak-vs-sanctioned load gauge, power-factor trend.

📞

1912 complaints

Live status of every complaint from your phone, with full JE/AE/XEN officer chain and one-click "Report outage" panel.

Quickstart

Python 3.10+ and Node 20+ (or Bun 1.1+) required.

# clone and set up
git clone https://github.com/Harry-kp/uppcl-pro.git
cd uppcl-pro
make setup       # Python + Node deps
make dev         # proxy :8000 + dashboard :3000
# first-time login
curl -X POST localhost:8000/auth/login \
  -H 'content-type: application/json' \
  -d '{"username":"<phone>","password":"<pw>"}'

# then open
open http://localhost:3000

Using Claude Code? Type /setup — skills are bundled in .claude/skills/.
